Offline Mode Concept

How LMEWS behaves when internet connectivity is lost

Internet Disconnected — Offline Mode Active
System running on cached data · Last sync: 6 minutes ago · SMS relay still functional
Cache age: 00:06:12
✓ What Works Offline
✓ Cached map tiles (last 72 hrs)
✓ Pre-loaded alert thresholds
✓ SMS sending via GSM (no internet)
✓ Local sensor data (river gauge, rain)
✓ Volunteer contact database
✓ Offline alert decision rules
✓ Voice broadcast system
✓ Community kiosk displays
✗ What Requires Internet
✗ Live satellite imagery (GEE)
✗ National weather API sync
✗ Real-time NDVI/NDWI updates
✗ Dashboard web interface
✗ Cloud data backup
✗ Remote volunteer app sync
NOTE: SMS alerts remain functional on 2G/3G GSM even when internet is down. This is the key design advantage.
Offline Alert Protocol (Decision Tree)
1
Local river gauge sensor detects water level {'>'} 8.5m threshold
→ Trigger cached flood alert rule (no internet needed)
2
LMEWS edge device auto-generates alert from cached rules
→ Alert classified: HIGH FLOOD RISK for Natore Sadar
3
SMS batch dispatched via local GSM modem (no internet)
→ 8,240 numbers receive: "সতর্কতা: বন্যার সম্ভাবনা..."
4
Volunteers receive SMS and activate door-to-door protocol
→ Color-coded printed cards distributed to households
5
When internet restored: sync all events to cloud dashboard
→ Full audit trail maintained for post-event analysis
